# Data Directory

This directory contains the project's datasets, including master registries, raw external data, and processed intermediate files.

## Directory Structure

### Top Level
- **`candidates_master.csv`**: The canonical master list of gene candidates for the project. This file is version-controlled and manually curated.
- **`datasets_registry.md`**: A registry of external datasets (e.g., OSD-665) used in the research.

### Subdirectories

#### `raw/`
Contains raw, unmodified data from external sources. These files are generally **not versioned** (ignored by git) due to size.
- **`afdb/`**: AlphaFold Protein Structure Database files (`.pdb`, `.cif`, `.json`).
  - *Note*: In offline mode, these are populated from the local `archive/` by `restore_local_data.py`.

#### `processed/`
Contains intermediate data generated by scripts from raw data or master lists. These files are **not versioned**.
- **`candidates.csv`**: A subset of `candidates_master.csv` filtered for data availability. Generated by `setup_available_data.py`.

## Workflows

### Data Restoration (Offline/Archive Mode)
To set up the data environment using local archived files:

1. **Setup Candidates**:
   ```bash
   python research/alphafold_countercurvature/scripts/setup_available_data.py
   ```
   This generates `data/processed/candidates.csv`.

2. **Restore PDBs**:
   ```bash
   python research/alphafold_countercurvature/scripts/restore_local_data.py
   ```
   This copies available PDB files from `archive/` to `data/raw/afdb/` and creates a manifest.
